# DESIGN

Дизайн-референс проекта. Бизнес/бренд-стратегия публичного сайта — в `PRODUCT.md`; здесь — реализация (токены и как стилизовать обе поверхности).

## Две поверхности

- **Публичный сайт** — Tailwind 4, исходник `resources/css/app.css` (блок `@theme` + готовые классы `.btn-primary`, `.btn-ghost`, `.card-steel`, `.eyebrow`, `.glass`). Шрифты: Sora (display) / Inter Tight / JetBrains Mono.
- **Админка (Filament v5)** — тема `resources/css/filament/admin/theme.css` (подключена через `->viteTheme(...)`). Шрифт Inter, primary Indigo. **Тему ниже НЕ наследует** — у админки своя.

## Тема публичного сайта: СВЕТЛАЯ «сталь» + бренд-цвета логотипа

Сайт — **светлый** (светло-стальной фон, белые карточки, тёмный текст). Акценты взяты из логотипа автосервиса: **синий `#0066B3` + красный `#ED1B2F`**. Источник правды — `@theme` в `resources/css/app.css`.

### Ключевой нюанс: шкала `ink` инвертирована под свет

Семантика классов прежняя (`bg-ink-900` = фон, `bg-ink-800` = карточка, `text-ink-100/200` = текст, `border-ink-700/600` = бордеры), но **значения перевёрнуты** — низкие номера тёмные (текст), высокие светлые (поверхности):

| Токен | Назначение | HEX |
|---|---|---|
| `ink-100` | заголовки (тёмный текст) | `#14181E` |
| `ink-200` | основной текст | `#2A323B` |
| `ink-300` | вторичный текст | `#434B55` |
| `ink-400` / `ink-500` | приглушённый / подписи | `#5C646E` · `#6E7681` |
| `ink-700` / `ink-600` | бордер / сильный бордер | `#E2E6EC` · `#CDD3DC` |
| `ink-800` | фон карточки (белый) | `#FFFFFF` |
| `ink-900` | основной фон (свет-сталь) | `#F4F6F9` |

| Акцент | 500 / 400 / 600 / 300 |
|---|---|
| `primary` (синий лого) | `#0066B3` · `#1E7BC0` · `#00538F` · `#4F9CD2` |
| `accent` (красный лого) | `#ED1B2F` · `#F0394A` · `#C71426` |

Красный — **точечно** (риска `.eyebrow::before`, акцент-точка на видео в hero). Не заливать им поверхности.

### Правила (важно соблюдать на светлой теме)

- **Белый текст (`text-white`) — только на синих/тёмных подложках** (кнопки `bg-primary-*`, скрим модалок, подписи поверх видео). На светлых поверхностях — `text-ink-100/200/300`.
- **Логотип** — реальный `storage/app/public/logo/logo.svg` (цветной), путь `asset('storage/logo/logo.svg')`. Используется в шапке, футере, прелоадере. Бейджа-монограммы «A» больше нет.
- **Декоративный блюр запрещён** (принципы impeccable): класс `.glow` отключён (`display:none`); без `backdrop-blur`-«стекла» по умолчанию; без `background-clip:text` градиент-текста (заголовки — сплошным цветом, акцент через `primary-500`).
- **Тёмные секции на светлом сайте** (если понадобятся) — оборачивать в `.on-dark` (определён в `app.css`): внутри он принудительно делает текст/кнопки/бордеры светлыми; для светлой карточки внутри такой секции есть исключение `.on-dark .card-steel ...`. Сейчас hero сделан светлым сплитом (видео в рамке-витрине справа), `.on-dark` свободен для будущего.
- **Проверка — `npm run build`** + `php artisan view:cache` (браузер в этой среде недоступен; `artisan serve` не биндится).

## Как стилизовать Filament

- Правим **только `theme.css`** — обычный CSS после `@import`. Blade/Livewire не трогаем (форма ломается, риск). Кастомный view страницы — лишь когда CSS не вытягивает (напр. сплит-экран).
- Утилиты Tailwind перебиваем через `!important`.

### Карта классов экрана входа (SimplePage)

Стили входа скоупим на `.fi-simple-layout` — основную панель не заденет. DOM:

```
.fi-simple-layout            ← фон страницы
 └ .fi-simple-main-ctn       ← центрирование
   └ .fi-simple-main         ← карточка
     └ .fi-simple-page
       └ .fi-simple-header   ← .fi-logo (brandName), .fi-simple-header-heading (h1), -subheading
       └ форма
```

Поля/контролы: `.fi-input-wrp` (рамка — фон + кольцо через `box-shadow: inset 0 0 0 1px`), `.fi-input` (цвет текста), `.fi-fo-field-wrp-label` (подпись), `.fi-btn` (кнопка отправки), `.fi-link`, `.fi-checkbox-input`.

Render-хуки для вставки разметки: `SIMPLE_LAYOUT_START/END`, `SIMPLE_PAGE_START/END`, `AUTH_LOGIN_FORM_BEFORE/AFTER`.
